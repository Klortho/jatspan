%all-phrase;

All Phrase-Level Elements

Definition

This Parameter Entity contains all of the phrase-level elements in the entire Archival Tag Set except for the <x> and <break> elements.

Remarks

This change results from the version 2.0 split of the NCBI Archiving and Publishing Tag Sets into three DTDs. The restrictions in the phrase-level content models were in place to help regularize the archive. However, they have no place in the more descriptive, open archival Tag Set that the Archiving (Green) Tag Set is becoming. The cleanest way to accomplish this change was to relax the fairly clean class/mix structure that the base Suite and the old Archiving Tag Set used and create one layer of “onion” entities. Thus, individual entities such as %emphasized-text; will continue to exist, but they will contain all the phrase-level elements, through the use of the %all-phrase; Parameter Entity.

Developer’s Note: At the moment, the <break> and <x> elements are not considered to be “ordinary” phrase-level elements and are not part of the %all-phrase; parameter entity.

Design Notes:

  • Since this parameter entity is used in other mixes, %all-phrase; must be declared first, before any other mix.
  • Since it acts as a class (i.e., is used in mixes), %all-phrase; does not start with an OR bar, as all other inline mixes do.

Declaration

<!ENTITY % all-phrase   "%address-link.class; | %article-link.class; |
                         %appearance.class; | %emphasis.class;  |
                         %inline-display.class; |
                         %inline-math.class; | %math.class; |
                         %phrase.class; | %simple-link.class; |
                         %subsup.class;"                             >
