Archiving and Interchange Suite Naming Rules

  1. All XML names are to be in lower case. (Names include element names, attribute names, entity names, and IDs.)
  2. When two or more words are concatenated into an element name, a hyphen will be placed between the whole words (e.g., <verse-group>, <article-title>, <word-count>).
  3. The typographic emphasis elements are spelled out in full — <bold>, <italic>, <underline>, etc. — not put in as values of an attribute on an element such as <emphasis>. (Superscript and subscript are not considered to be purely typographic.)
  4. The casing and interior punctuation of element type names and attribute names inherited from PUBLIC models (such as the XHTML table model or the MathML Tag Set) are unchanged. Thus, for example, the Horizontal Rule element is <hr>, and the table cell tags are <td>, <tr>, <tbody>, <thead>, etc.
  5. Abbreviations may be used when words are used in combination. When a word stands alone as a name, it is not abbreviated. Thus, the element Conference Information uses the full name <conference>, but the conference combinations (such as Conference Theme and Conference Number) use the abbreviation “conf” (<conf-theme> and <conf-num>).
