NLM JATS Release Procedure

This documents the procedure to be followed to create a new release version.

  1. In the trunk directory, make sure that the tests pass successfully.
  2. Edit ReleaseNotes.txt.
    • Decide on a major/minor version number. Create a name for this release of the form "nlm-jats-v.vv", where v.vv is the version number.
    • Add a new section at to the top. Copy the format of previous sections.
  3. From trunk, commit everything, with a note about the release. E.g. 
    svn commit -m "Release v.vv, candidate 1"
    Increment the candidate number if this isn't the first attempt at a release for this version number.
  4. Create a tag in Subversion for this release, with a command similar to the following. 
    svn copy --parents \
  https://nlm-jats.svn.sourceforge.net/svnroot/nlm-jats/trunk \
  https://nlm-jats.svn.sourceforge.net/svnroot/nlm-jats/releases/v.vv \
  -m "Tagging release 'v.vv'."
    Verify that the release tag was created, by checking the respository. (See this page in the Subversion documentation for a discussion of using tags for releases.)
  5. Create the download file. From the trunk directory, enter a command similar to the following. This uses the InfoZip's zip utility. 
    zip -9r -ll nlm-jats-v.vv.zip nlm-jats -x *.svn*
    • v.vv is the version number of the release.
    • "-9" says to "compress better".
    • "-r" means recurse into subdirectories.
    • "-ll" causes all Windows line-endings 'CR LF' to be converted to Unix style 'LF'.
    • "-x *.svn*" excludes the subversion repository directories.
  6. Test one more time. This is important to ensure the integrity of the release file.
    • Unzip it to a temporary directory.
    • Rerun the tests.
  7. Rename the zip file to include the final Subversion repository revision. Use the revision number that was generated when you created the release tag (which you can see here). For example, "nlm-jats-1.16-r1024.zip".
  8. Upload the Zip file to the site. Also, if there were any changes to the README.txt file, upload it too, so that they are reflected on the Files page.
